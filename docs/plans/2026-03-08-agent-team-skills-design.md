# Agent Team Skills — Design

**Goal:** A skills package that extends obra/superpowers with team-based implementation: named Developer+Tester and Architect agents, state tracking for resumability, and orchestrator progress display.

**Architecture:** Orchestrator skills delegate to obra skills internally, adding a team coordination layer on top. Two new execution roles (Developer, Architect) are dispatched as named agents. State is persisted in `.state.json` alongside the plan file, with checkboxes embedded in the plan for human readability.

**Tech Stack:** `npx skills` (skills.sh), obra/superpowers skills as internal delegates, SKILL.md format, JSON state file, git worktrees.

## Skills

### `team-plan`

Wraps obra `writing-plans`. After plan is generated:
1. Adds `## Progress` section with task checkboxes (`⏳ pending`)
2. Adds resume header: `> **Resume:** Check \`*.state.json\` for current progress`
3. Creates initial `.state.json` with all tasks at `pending`

### `team-execute`

Main orchestrator. On start:
1. Reads plan + `.state.json` (skips completed tasks on resume)
2. Generates and displays **resume prompt** (plan path, state path, worktree, branch, next task)
3. Per task loop:
   - Dispatches **Developer agent** (`team-developer`) with full task text + context
   - Prints progress checklist after status change (`✅ / 🔄 / ⏳`)
   - Updates `.state.json` (worktree, branch, commit SHA, timestamps)
   - Developer agent pushes commit to remote after task
   - Dispatches **Architect agent** (`team-architect`) with diff + plan + ADRs + design doc
   - If CHANGES_REQUIRED: Developer fixes → Architect re-reviews
   - On APPROVED: marks task complete in checklist + `.state.json`, pushes
4. After all tasks: deletes `.state.json`, commits deletion, pushes
5. Calls obra `finishing-a-development-branch`

### `team-developer`

Agent role — receives: task text, context, worktree path, branch.
- Uses obra `using-git-worktrees` for isolation
- Works in TDD mode (obra `test-driven-development`)
- Pushes commit to remote after completing task
- Writes result to `.state.json`: commit SHA, test results, worktree path

### `team-architect`

Agent role — receives: git diff, task spec, ADRs, design doc, project conventions.
- Uses obra `requesting-code-review`
- Returns: `APPROVED` or `CHANGES_REQUIRED` with specific issues
- Runs after Developer completes each task

## State Schema (`.state.json`)

```json
{
  "plan": "docs/plans/2026-03-08-feature.md",
  "worktree": "/path/to/.worktrees/feature-branch",
  "branch": "feature/my-feature",
  "updated": "2026-03-08T14:23:00Z",
  "resumePrompt": "Resume: plan at docs/plans/..., next task #2, worktree /path/...",
  "tasks": [
    {
      "id": 1,
      "title": "Setup auth module",
      "status": "completed",
      "agent": "developer",
      "commit": "abc1234",
      "worktree": "/path/to/.worktrees/feature-branch",
      "completedAt": "2026-03-08T13:00:00Z"
    },
    {
      "id": 2,
      "title": "Add JWT validation",
      "status": "in_progress",
      "agent": "developer",
      "worktree": "/path/to/.worktrees/feature-branch",
      "startedAt": "2026-03-08T14:20:00Z"
    }
  ]
}
```

## Resume Prompt (generated by `team-execute`)

```
Resume implementation:
- Plan: docs/plans/2026-03-08-feature.md
- State: docs/plans/2026-03-08-feature.state.json
- Worktree: /path/.worktrees/feature-branch (branch: feature/my-feature)
- Next task: #2 "Add JWT validation"

Use team-execute skill to continue.
```

Shown to user before start and saved in `.state.json`. Paste into new session to resume.

---

## Cleanup

- `.state.json` is listed in `.gitignore` of the target project during work
- Before `finishing-a-development-branch`: orkiestrator deletes `.state.json`, commits, pushes
- Agents push after every completed task (feature branch, not main)
